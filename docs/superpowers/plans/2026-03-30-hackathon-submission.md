## Task 1: Backtest Engine

**Files:**
- Create: `~/local-dev/nanuqfi-keeper/scripts/backtest.ts`
- Output: `~/local-dev/nanuqfi-app/src/data/backtest-results.json`

- [ ] **Step 1: Create backtest script skeleton**

Create `scripts/backtest.ts` in the keeper repo. Import `AlgorithmEngine` from `src/engine/algorithm-engine.ts`. Define the output JSON shape interface matching the spec. Set up the main function that:
1. Fetches 90 days of historical Drift rates from the Drift Data API
2. For each day, runs `AlgorithmEngine.proposeWeights()` for moderate and aggressive
3. Calculates daily portfolio return as weighted sum of strategy returns
4. Tracks a USDC lending baseline for comparison
5. Computes metrics: APY, max drawdown, Sharpe ratio, Sortino ratio, win rate, alpha over baseline
6. Writes output JSON to `../nanuqfi-app/src/data/backtest-results.json`

Key implementation details:
- Use `AlgorithmEngine` constructor from `src/engine/algorithm-engine.ts` — it takes `riskLevel` and has `proposeWeights(yieldData)` method
- `YieldData` shape: `{ usdcLendingRate, solFundingRate, solFundingHistory, solBorrowRate, jitoStakingYield }`
- Strategy returns per day: lending = `lendingRate / 365`, basis = `|fundingRate| / 365` (delta-neutral capture), funding = `fundingRate / 365` (directional), jitodn = `(jitoStakingYield - solBorrowRate) / 365`
- Auto-exit triggers: if basis spread < 0 for the day, basis weight goes to 0 (simplified daily proxy for the 4h trigger). If funding is negative, funding weight goes to 0.
- Sharpe = `(mean(dailyReturns) / std(dailyReturns)) * sqrt(365)`
- Sortino = `(mean(dailyReturns) / downside_std(dailyReturns)) * sqrt(365)` where downside_std only counts negative returns
- Output path configurable via CLI arg, defaults to `../nanuqfi-app/src/data/backtest-results.json`

For Drift Data API historical data: use `https://data.api.drift.trade/fundingRates?marketIndex=0` and `https://data.api.drift.trade/spotMarketHistory?marketIndex=0`. If these don't have full 90-day history, fall back to the DeFi Llama yields API (`https://yields.llama.fi/chart/<pool-id>`) for Drift USDC lending pool.

- [ ] **Step 2: Create `use-in-view` hook**

Create `src/hooks/use-in-view.ts`:

```ts
'use client'
import { useEffect, useRef, useState } from 'react'

export function useInView(options?: IntersectionObserverInit) {
  const ref = useRef<HTMLDivElement>(null)
  const [inView, setInView] = useState(false)

  useEffect(() => {
    const el = ref.current
    if (!el) return

    const observer = new IntersectionObserver(
      ([entry]) => { if (entry.isIntersecting) setInView(true) },
      { threshold: 0.1, ...options }
    )
    observer.observe(el)
    return () => observer.disconnect()
  }, [options])

  return { ref, inView }
}
```

Triggers once (no un-trigger on scroll out) — appropriate for fade-in animations.

```tsx
'use client'
import { useInView } from '@/hooks/use-in-view'

export function FadeIn({
  children,
  className = '',
  delay = 0,
}: {
  children: React.ReactNode
  className?: string
  delay?: number
}) {
  const { ref, inView } = useInView()

  return (
    <div
      ref={ref}
      className={`transition-all duration-700 ease-out ${
        inView ? 'opacity-100 translate-y-0' : 'opacity-0 translate-y-8'
      } ${className}`}
      style={{ transitionDelay: `${delay}ms` }}
    >
      {children}
    </div>
  )
}
```

- [ ] **Step 4: Create `AnimatedCounter`**

Create `src/components/pitch/AnimatedCounter.tsx`:

```tsx
'use client'
import { useEffect, useState } from 'react'
import { useInView } from '@/hooks/use-in-view'

export function AnimatedCounter({
  value,
  suffix = '',
  prefix = '',
  decimals = 1,
  duration = 1500,
  className = '',
}: {
  value: number
  suffix?: string
  prefix?: string
  decimals?: number
  duration?: number
  className?: string
}) {
  const { ref, inView } = useInView()
  const [display, setDisplay] = useState(0)

  useEffect(() => {
    if (!inView) return
    const start = performance.now()
    const animate = (now: number) => {
      const elapsed = now - start
      const progress = Math.min(elapsed / duration, 1)
      const eased = 1 - Math.pow(1 - progress, 3) // ease-out cubic
      setDisplay(value * eased)
      if (progress < 1) requestAnimationFrame(animate)
    }
    requestAnimationFrame(animate)
  }, [inView, value, duration])

  return (
    <span ref={ref} className={`font-mono tabular-nums ${className}`}>
      {prefix}{display.toFixed(decimals)}{suffix}
    </span>
  )
}
```

```ts
interface ChartLine {
  data: number[]       // y values (cumulative returns)
  color: string        // tailwind color like '#38bdf8' (sky-400)
  dashed?: boolean     // for baseline
  label: string        // legend label
}

interface SvgLineChartProps {
  lines: ChartLine[]
  labels: string[]     // x-axis labels (dates)
  height?: number      // default 300
  className?: string
}
```

Implementation:
- `viewBox` based on data range. X-axis evenly spaced, Y-axis auto-scaled to min/max across all lines.
- Each line is a `<path>` with `d` attribute built from `M x0,y0 L x1,y1 L x2,y2 ...`
- Dashed lines use `stroke-dasharray="6,4"`
- Y-axis: a few horizontal gridlines with labels (percentage)
- X-axis: show every ~15th date label to avoid crowding
- SVG `preserveAspectRatio="none"` with percentage-based width for responsiveness
- Legend below chart: colored dot + label for each line

MVP: no hover tooltips. Just the paths + labels + legend.
